# Size Specification

## Universal AI Symbol

**Document:** SIZES.md

**Version:** 1.0

**Status:** Draft

---

# Purpose

This document defines size and scalability principles for the Universal AI Symbol.

The objective is to ensure that the symbol remains recognizable and consistent across different physical and digital applications.

---

# Primary Principle

The Universal AI Symbol should preserve recognition regardless of scale.

The symbol should work from small digital applications to large physical installations.

Size should change the presentation of the symbol, not its identity.

---

# Minimum Usage Size

The minimum usable size depends on the application environment.

The symbol should remain recognizable when used in:

* application icons
* user interfaces
* documents
* printed materials
* educational resources
* physical objects

If reducing the symbol causes the geometry to become unclear, the size is too small.

---

# Digital Applications

The symbol should remain clear in:

* websites
* mobile interfaces
* software interfaces
* digital documents
* profile images

Small digital uses should preserve:

* recognizable silhouette
* clear contrast
* simple geometry

---

# Print Applications

The symbol should remain effective in:

* papers
* books
* posters
* signs
* educational materials

Printed versions should maintain:

* visual balance
* clean edges
* consistent proportions

---

# Large Scale Applications

The symbol should support large applications including:

* signage
* exhibitions
* physical displays
* architectural environments

Large-scale reproduction should preserve the original geometry without unnecessary modifications.

---

# Scaling Rules

When resizing the symbol:

Maintain:

* original proportions
* original geometry
* consistent shape relationships

Do not:

* stretch horizontally
* stretch vertically
* distort curves
* modify structural elements

---

# Recognition Test

A correctly scaled symbol should remain identifiable when viewed:

* from a distance
* at small size
* without additional explanation

Recognition should come from the symbol itself.

---

# Future Refinement

Future versions may define:

* official minimum dimensions
* pixel recommendations
* print measurements
* manufacturing requirements

These values should be based on finalized geometry and practical testing.

---

# Principle

A universal symbol should not depend on scale.

Its identity should remain present from the smallest representation to the largest application.
